🔒 Privacy and Security Analysis

1. Data Persistence and Privacy

In accordance with the assessment requirements, this inventory application operates entirely on the client side. No user data or inventory records are transmitted to an external server. All data is temporarily stored within the browser's active memory (session state) using Angular Services. Once the browser tab is closed, the data is completely erased, ensuring zero risk of persistent data breaches or unauthorized third-party access.

2. Input Validation (Data Integrity)

To prevent database corruption and logic errors, strict input validation is enforced across all forms:

  • Type Checking: Numeric fields (like Quantity and Price) utilize strict HTML5 type="number" attributes and TypeScript logic to reject alphabetical characters.
  • Uniqueness Enforcement: The system actively checks the Item ID against existing records to prevent duplication.
  • Required Fields: Form submissions are blocked and error messages are displayed if mandatory fields are left blank.

3. Cross-Site Scripting (XSS) Prevention

This application leverages Angular's built-in security mechanisms. Angular treats all values as untrusted by default. When values are inserted into the DOM (e.g., displaying an item name), Angular automatically sanitizes and escapes the content. This prevents malicious users from injecting executable scripts into the inputs (XSS attacks), making the UI secure by design.

4. Future Security Considerations (If Scaling to Backend)

While this prototype is secure as a standalone client app, connecting it to a real server in the future would require:

  • Implementing JWT (JSON Web Tokens) for secure user authentication and route guarding.
  • Enabling HTTPS/TLS encryption for data in transit.
  • Adding server-side validation, as client-side validation can be bypassed by malicious actors interacting directly with the API.
